## 1. 결론 요약

**MS-8은 기존 MS-2에서 부분 구현된 월별 관리 기능을 UI로 전면화하는 단계입니다.**

- 🎯 안전 수준: **높음** (새 데이터 추가 없음, 기존 기능 활용만)

## 2. MS-8 공식 정의 정리

### 2.1 목적
**업로드 이력과 월별 데이터 운영 편의성을 높인다.**

### 2.2 포함 기능

| 기능 | 설명 |
|------|------|
| 업로드 월 목록 | 저장된 모든 월(year-month)의 파일 목록 표시 |
| 파일명 / 업로드일 / 행 수 | 각 월별 파일의 메타데이터 표시 |
| 삭제됨 로그 확인 | 삭제된 월의 기록 및 삭제 시간 표시 |
| 월별 재업로드 진입 | 특정 월을 다시 업로드하기 위해 업로드 화면으로 이동할 수 있는 진입 UX (단순 화면 이동만, 자동 삭제/덮어쓰기 없음) |
| 이슈 수 표시 | 해당 월의 미매칭 이슈 건수 표시 |

### 2.3 제외 기능

- ❌ 월별 데이터 자동 재분류
- ❌ 복구 기능 (삭제된 월 되돌리기)
- ❌ 과거 모든 월 일괄 조회/처리
- ❌ 단순 파일 목록 이상의 고도화된 분석 UI
- ❌ **월별 재업로드 자동화** (삭제 후 자동 덮어쓰기, 자동 저장 등)
  - MS-8의 "재업로드 진입"은 업로드 화면 이동만 의미
  - 사용자가 수동으로 기존 월 삭제 → 새 파일 업로드해야 함 (MS-2 정책 준수)

### 2.4 선행 단계와의 연결

```
MS-2: 업로드 저장 + 중복 월 + 삭제 UX
  ↓ (기반)
MS-8: 월별 데이터 관리 고도화
  ├─ MS-2의 중복 월 감지 재활용
  ├─ MS-2의 삭제 로직 재활용
  └─ MS-2가 저장한 sales_files, upload_log 조회
```

### 2.5 MS-9 보고서센터와의 경계

**MS-9는 월별 데이터를 기반으로 보고서(보여주기)를 생성**:
- 판매량 보고서
- 판매량 비교 보고서

**MS-8은 월별 데이터 관리(저장/삭제/조회)**:
- 업로드 이력 조회
- 월 데이터 정리

→ **책임 분리**: MS-8은 데이터 관리, MS-9는 데이터 활용
