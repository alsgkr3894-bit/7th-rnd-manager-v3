## 2. MS-7.1 중복 등록 방지 처리 방식

### 원리
저장 버튼(onclick 핸들러)에서 데이터 입력 검증 다음에 **중복 확인 로직**을 추가했습니다.

```javascript
// 기존 검증 후
if (!rawName || !mappedName) { ... }

// 새로운 중복 확인
const duplicate = aliases.find(a => a.rawName === rawName);
if (duplicate) {
  toast.warn(`"${rawName}"은(는) 이미 등록된 별칭입니다.`);
  return;  // 저장 중단
}

// 이후 저장 진행
await put('ref_sales_aliases', { rawName, mappedName, enable: true });
```

### 중복 차단 기준

#### 별칭 (ref_sales_aliases)
- **비교 대상**: `rawName` (정확 일치)
- **검색 대상**: 현재 메모리 로드된 `aliases` 배열
- **메시지**: `"[rawName]"은(는) 이미 등록된 별칭입니다.`
- **차단 범위**: 활성/비활성 상관없이 모든 별칭 검사

**코드 위치**: 라인 378-380

#### 제외 항목 (ref_excluded)
- **비교 대상**: `menuName` (정확 일치)
- **검색 대상**: 현재 메모리 로드된 `excluded` 배열
- **메시지**: `"[menuName]"은(는) 이미 등록된 제외 항목입니다.`

**코드 위치**: 라인 429-431

#### 분류 규칙 (sales_rules)
- **비교 대상**: `rawMenuName` (정확 일치)
- **검색 대상**: 현재 메모리 로드된 `rules` 배열
- **메시지**: `"[rawMenuName]"은(는) 이미 등록된 분류 규칙입니다.`
- **차단 범위**: 활성 규칙만 검사 (비활성 제외, loadRules에서 `enable !== false` 필터링)

**코드 위치**: 라인 497-499

## 4. MS-7.2 삭제 전 확인 모달 처리 방식

### 모달 구조

```
사용자 삭제 버튼 클릭
  ↓
deleteItem(type, id) 호출
  ↓
showDeleteConfirmModal() - 모달 생성 및 표시
  ↓
  ┌─ 취소 버튼 클릭 → hideDeleteConfirmModal() → 아무것도 안 함
  │
  └─ 삭제 버튼 클릭 → performDelete(type, id) → 실제 삭제 실행 → 목록 새로고침
```

### 모달 UI 요소

**제목**: `⚠️ 삭제 확인` (경고 아이콘)

**삭제 대상 표시**:
```
<strong>제외 항목:</strong> [대상명]
```
(배경: 밝은 회색 #f9f9f9, 왼쪽 테두리: 빨간색 3px)

**영향 안내**:
```
향후 업로드 자동 반영 기준에서 제거됩니다.
```
(색상: 회색 #666, 크기: 13px)

**버튼**:
- "취소" (회색): 모달만 닫음
- "삭제" (빨간색 btn-danger): 실제 삭제 실행

### 코드 위치

- `showDeleteConfirmModal()`: 라인 244-296
- `hideDeleteConfirmModal()`: 라인 298-304
- `performDelete()`: 라인 306-325
- `deleteItem()` (호환성 함수): 라인 327-329

## 5. 삭제 확정/취소 흐름

### 흐름도

```
Step 1: 삭제 버튼 클릭
  ↓
Step 2: showDeleteConfirmModal(type, id) 실행
  - 대상 항목 찾기 (excluded.find())
  - 모달 DOM 생성 또는 기존 DOM 재사용
  - 모달 UI 렌더링 (제목, 대상 정보, 메시지, 버튼)
  - backdrop.display = 'block'
  - panel.display = 'block'
  ↓
Step 3: 사용자 선택
  ├─ "취소" 클릭
  │   ↓
  │   hideDeleteConfirmModal()
  │     - backdrop.display = 'none'
  │     - panel.display = 'none'
  │   ↓
  │   [종료] (데이터 유지, 아무 변화 없음)
  │
  └─ "삭제" 클릭
      ↓
      hideDeleteConfirmModal() 실행 (모달 즉시 닫음)
      ↓
      performDelete(type, id) 실행
        - deleteById() 호출 (DB 실제 삭제)
        - loadExcluded() 또는 loadAliases() 호출 (목록 새로고침)
        - toast.success('삭제되었습니다.') 표시
      ↓
      [종료] (데이터 삭제, 목록 반영)
```

### 취소 시 동작
- 모달만 닫힘
- 기존 목록 유지
- 아무 DB 작업 없음

### 삭제 확정 시 동작
- 모달 닫힘
- `deleteById()` 실행 (type === 'excluded' ? 'ref_excluded' : 'ref_sales_aliases')
- 목록 새로고침 (`loadExcluded()` 또는 `loadAliases()`)
- 성공 토스트 메시지 표시
