# 7rnd v3 실데이터 QA 체크리스트

이 문서는 안정화 커밋 `672fb18 chore: harden site stability guards` 이후 실업무 데이터로 확인해야 할 항목을 정리한다.  
새 데이터를 임의 생성하거나 계산식, 저장/삭제, 업로드/다운로드, 엑셀 출력 형식을 바꾸지 않고 검증한다.

## 현재 확인된 fixture

- `public/templates/원산지_템플릿.xlsx`
  - 시트: `원산지정보`, `원산지표지판`, `원산지표지판 (냉장고부착용)`
  - 회귀 테스트: `__tests__/lib/origin-template-fixture.test.mjs`
- `lib/ingredient/master-seed.js`
  - 제품코드 기반 식자재 마스터 seed 80건
  - 실데이터 제품코드 매칭 비교의 기준 후보로 사용 가능

판매량, 제때 단가, 원가 계산 결과, 메뉴코드 매칭용 실업무 엑셀 fixture는 현재 별도 파일로 확인되지 않았다. 실제 QA에서는 아래 형식의 업무 파일을 준비한다.

## 준비할 실데이터 파일

### 메뉴 판매량 업로드

- 파일: `.xlsx`, `.xls`, `.csv`
- 필수 정보:
  - 판매 기간이 파일 상단 20행 이내에서 탐지 가능해야 한다.
  - 헤더는 상단 20행 이내에 있어야 한다.
  - 메뉴명 헤더: `메뉴명` 또는 `메뉴 명`
  - 판매량 헤더: `판매량(개)` 또는 `판매량 (개)`
- 정상 기대:
  - 모든 판매량은 0 이상 안전 정수다.
  - 같은 월 중복 업로드는 차단된다.
  - 미매칭 메뉴는 미매칭 목록으로 분리된다.

### 제때 단가 업로드

- 파일: `.xlsx`, `.xls`, `.csv`
- 필수 컬럼:
  - 제품명: `제품명`, `상품명`, `품목명`
  - 과세구분: `과세구분`, `과세`, `면세`
  - 단가: `단가`, `신규 판매가`, `판매가`
- 권장 컬럼:
  - 제품코드: `제품코드`, `상품코드`, `코드`
  - 제품상태, 온도, 판매단위, 수량, 합계금액, 공급가, 부가세, 주문기준일, 배송일
- 정상 기대:
  - 과세 품목은 단가에 부가세 반영 금액이 표시된다.
  - 동일 날짜 또는 동일 파일 내용 중복 업로드가 차단된다.
  - 제품코드가 있는 행은 제품코드 기준으로 비교된다.

### 메뉴 판매가/메뉴 마스터

- 메뉴 판매가 필수 컬럼:
  - 메뉴명: `메뉴명`, `상품명`, `메뉴이름`
  - 판매가: `판매가`, `가격`, `판매가격`
- 권장 컬럼:
  - 메뉴코드: `메뉴코드`, `코드`
  - 분류, 규격, 비고
- 정상 기대:
  - 메뉴코드는 메뉴 마스터와 일치한다.
  - 가격이 비어 있거나 숫자로 인식되지 않는 행은 실패로 분리된다.
  - 메뉴 마스터 CSV 다운로드의 컬럼 순서는 `메뉴코드`, `메뉴명`, `판매가`, `상태`, `카테고리`다.

### 식자재 마스터/원가 계산

- 식자재 권장 컬럼:
  - 제품코드, 제품명, 과세구분, 기준수량, 기준단위, 단가, 분류, 태그
- 정상 기대:
  - 제품코드 기준으로 기존 식자재가 업데이트된다.
  - 조합 식자재의 참조 제품코드는 누락 여부를 사람이 확인한다.
  - 원가 계산 결과는 기존 기준 파일 또는 전월 산출물과 숫자로 비교한다.

## 회귀 확인 절차

1. `npm run lint`, `npm test`, `npm run build`, `npm run qa:smoke`를 실행한다.
2. 실데이터 업로드 전 브라우저에서 백업을 만든다.
3. 테스트용 데이터 또는 복구 가능한 업무 데이터로만 업로드/삭제/복원을 확인한다.
4. 업로드 후 화면 표시값과 다운로드 파일 값을 원본 파일과 비교한다.
5. 원가 계산 결과는 기준표의 메뉴코드별 값과 비교한다.

## 실데이터 QA 체크리스트

### 엑셀 업로드

- [ ] 메뉴 판매량 실제 파일 업로드가 성공한다.
- [ ] 메뉴 판매량 파일의 기간이 올바르게 감지된다.
- [ ] 메뉴 판매량 중복 월 업로드가 차단된다.
- [ ] 제때 단가 실제 파일 업로드가 성공한다.
- [ ] 제때 단가 중복 날짜 또는 동일 파일 업로드가 차단된다.
- [ ] 잘못된 확장자 파일이 거부된다.
- [ ] 필수 컬럼 누락 파일에서 명확한 에러 메시지가 표시된다.
- [ ] 빈 파일 또는 데이터 행 없는 파일에서 명확한 에러 메시지가 표시된다.

### 엑셀/CSV 다운로드

- [ ] 메뉴 마스터 CSV 다운로드 컬럼과 순서가 유지된다.
- [ ] 원가 세부표 CSV 다운로드 컬럼과 순서가 유지된다.
- [ ] 영양성분 엑셀 다운로드가 성공한다.
- [ ] 원산지 엑셀 다운로드가 성공한다.
- [ ] 다운로드 파일을 다시 열었을 때 한글, 줄바꿈, 숫자 형식이 깨지지 않는다.

### 백업/복원

- [ ] 설정/백업에서 JSON 백업 파일이 생성된다.
- [ ] 설정/복원에서 정상 백업 파일을 읽을 수 있다.
- [ ] 잘못된 백업 파일은 복원 전에 차단된다.
- [ ] 복원 후 주요 화면 데이터 수와 대표 값이 복원 전 기준과 일치한다.

### 저장/수정/삭제

- [ ] 메뉴 마스터 신규 저장이 성공한다.
- [ ] 메뉴 마스터 수정이 성공한다.
- [ ] 메뉴 마스터 삭제 또는 비활성 상태 처리가 의도대로 동작한다.
- [ ] 식자재 수동 등록 저장이 성공한다.
- [ ] 식자재 수정이 성공한다.
- [ ] 삭제 가능 항목과 숨김 처리 항목이 구분된다.
- [ ] 메뉴판매량 업로드 이력 삭제가 테스트 데이터에서만 정상 동작한다.
- [ ] 제때 단가 파일 삭제가 테스트 데이터에서만 정상 동작한다.

### 검색/필터/정렬/페이지네이션

- [ ] 메뉴 마스터 검색이 메뉴명과 메뉴코드 기준으로 동작한다.
- [ ] 식자재 검색이 제품명과 제품코드 기준으로 동작한다.
- [ ] 메뉴 판매량 필터가 분류/기간 기준으로 동작한다.
- [ ] 원가 테이블 정렬이 숫자와 문자열 모두 올바르게 동작한다.
- [ ] 페이지네이션에서 첫 페이지, 중간 페이지, 마지막 페이지 이동이 동작한다.
- [ ] 빈 결과에서 페이지네이션이 깨지지 않는다.

### 원가 계산 결과

- [ ] 대표 피자 메뉴 3개 이상을 기준표와 비교한다.
- [ ] 사이드 메뉴 3개 이상을 기준표와 비교한다.
- [ ] 세트/1인피자/엣지·도우 항목을 각각 1개 이상 비교한다.
- [ ] 원가율 계산이 `원가 / 판매가 * 100` 기준과 일치한다.
- [ ] 판매가가 0 또는 비어 있는 항목에서 원가율 표시가 깨지지 않는다.
- [ ] 제때 단가 변경 후 영향받는 메뉴의 원가가 예상 범위로 갱신된다.

### 코드 매칭/중복 검증

- [ ] 메뉴코드가 메뉴 마스터의 실제 메뉴와 일치한다.
- [ ] 메뉴코드 중복 입력이 차단된다.
- [ ] 제품코드가 제때 단가/식자재 마스터와 일치한다.
- [ ] 제품코드 중복 입력 또는 중복 업로드가 차단된다.
- [ ] 판매량 미매칭 메뉴가 미매칭 화면에 표시된다.
- [ ] 사용자 별칭/분류 룰 중복 등록이 차단된다.

### 화면/상태

- [ ] 모바일 390px 폭에서 주요 화면에 가로 overflow가 없다.
- [ ] 태블릿 702px 폭에서 smoke 대상 22개 화면에 가로 overflow가 없다.
- [ ] 데스크톱 폭에서 카드, 버튼, 테이블 헤더 정렬이 깨지지 않는다.
- [ ] 업로드/저장/삭제 중 로딩 상태가 표시된다.
- [ ] 실패 시 에러 메시지가 사용자에게 표시된다.
- [ ] 성공 시 toast 또는 상태 메시지가 표시된다.

## 배포 전 확인

- [ ] 테스트 데이터와 실제 업무 데이터를 구분했다.
- [ ] 백업 파일을 먼저 생성했다.
- [ ] 삭제/복원 테스트는 운영 데이터가 아닌 복구 가능한 데이터로만 수행했다.
- [ ] 원가 계산 기준표를 담당자가 확인했다.
- [ ] 다운로드 파일을 실제 Excel 앱에서 열어 확인했다.
- [ ] 최종 검증 명령이 모두 통과했다.
